# lls content view plan

`lls cat <path>` のざっくりした実装計画メモ。

## 目的

- ファイルの中身を確認できるようにする
- LLM や人間が内容を読めるが、危ない文字列は自動で隠す
- `ls` 系の探索フローから、1 段深い確認へ進めるようにする

## 範囲

- 対象は主にテキストファイル
- バイナリは原則全文表示しない
- デフォルトで redaction を有効にする
- `--raw` は明示オプトインにする
- 行番号付き表示を基本にする

## 実装の順番

1. `lls cat <path>` の引数解析を足す
2. ファイル種別とサイズを見て、テキスト/バイナリを分ける
3. 先頭から安全に読み、行番号付きで表示する
4. redaction ルールを入れる
5. `--raw` や `--head` / `--tail` を必要なら足す
6. 結果を `--human` と統一感のある見せ方に寄せる

## redaction 方針

- API key っぽい値を隠す
- Bearer token を隠す
- JWT っぽい長いトークンを隠す
- private key のヘッダ/フッタを隠す
- `.env` 形式の秘密っぽい値を隠す
- 誤検知より漏えい防止を優先する

## テスト観点

- テキストファイルが読めること
- バイナリを誤って全文表示しないこと
- redaction が効くこと
- `--raw` で生表示できること
- 大きすぎるファイルで安全側に倒れること
- 行番号や表示形式が崩れないこと

## 先送りしてよいもの

- 完全な秘密情報検出
- 高度な言語別の色分け
- すべてのファイル形式への対応
- LLM への最適化を極めたプロンプト整形
